# room_reservation (OfficeNomad)

## Система бронирования переговорных комнат
Одна большая IT-компания построила крупное офисное здание, и в скором времени в нём будут работать тысячи людей. 

В здании, помимо офисов, будет много переговорных комнат. Сотрудники любого отдела компании cмогут бронировать эти комнаты и проводить там встречи, планёрки, симпозиумы и прочие брейнстормы с вечеринками. 

Чтобы не было конфликтов и не срывались планы — нужен простой и прозрачный механизм, который позволил бы сотрудникам резервировать переговорки на определённый период времени. 

Ваша задача — написать API для приложения OfficeNomad; через это приложение сотрудники будут бронировать помещения. В дальнейшем компания планирует продавать приложение OfficeNomad как часть программного обеспечения для офисных центров. 

Предполагается, что приложением будет пользоваться множество пользователей одновременно, и чтобы запросы обрабатывались без задержки, приложение должно быть асинхронным — оно должно работать не просто быстро, а очень быстро!

У пользователя должна быть возможность забронировать свободное помещение на определённый период времени, при этом приложение должно проверять, не забронировано ли это помещение на заданный период. 


В приложении должно быть три модели: 

- модель, описывающая переговорки;
- модель, описывающая бронирования (какая переговорка забронирована, кем и на какой период времени);
- модель пользователей (с разделением ролей на обычных пользователей и админов системы).
